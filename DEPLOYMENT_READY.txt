🚀 QUICK START - DEPLOY NOW
═══════════════════════════════════════════════════════════════════════════════════

Step 1: Deploy Backend to Render (5 minutes)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. Go to https://render.com and sign up/login
  2. Create PostgreSQL database named "ecomzone"
  3. Create Web Service and connect your GitHub repo
  4. Set environment variables (see DEPLOYMENT_GUIDE.md)
  5. Deploy!
  
  Expected Backend URL: https://ecomzone-backend-xxxx.onrender.com

Step 2: Deploy Frontend to Vercel (3 minutes)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. Go to https://vercel.com and sign up/login
  2. Import your GitHub repo
  3. Set root directory: frontend
  4. Add environment variables with backend URL
  5. Deploy!
  
  Expected Frontend URL: https://ecomzone-xxxx.vercel.app

Step 3: Configure CORS (2 minutes)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. Go to Render backend settings
  2. Update FRONTEND_URL with your Vercel URL
  3. Redeploy backend
  4. Done!

📝 ENVIRONMENT VARIABLES YOU'LL NEED
═══════════════════════════════════════════════════════════════════════════════════

For Render Backend:
  NODE_ENV=production
  PORT=3001
  DATABASE_URL=postgresql://user:password@host:port/database
  JWT_SECRET=<generate-random-string>
  FRONTEND_URL=https://your-vercel-url.vercel.app

For Vercel Frontend:
  NEXT_PUBLIC_BACKEND_URL=https://your-render-url.onrender.com
  NEXT_PUBLIC_API_URL=https://your-render-url.onrender.com/api

See DEPLOYMENT_GUIDE.md for detailed instructions on where to set these.

❓ FREQUENTLY ASKED QUESTIONS
═══════════════════════════════════════════════════════════════════════════════════

Q: How long does deployment take?
A: Render: 3-5 minutes | Vercel: 2-3 minutes | Total: ~10 minutes

Q: Do I need to pay?
A: Both Render and Vercel have free plans suitable for starting out.
   Render free tier: 750 hours/month, PostgreSQL 256MB
   Vercel free tier: Unlimited deployments, 100GB bandwidth

Q: Can I use a custom domain?
A: Yes, both platforms support custom domains. Configure in their dashboards.

Q: What if the database connection fails?
A: Check DATABASE_URL in Render settings. Try the connection string locally first.

Q: How do I update my code after deployment?
A: Push to GitHub → Both platforms auto-redeploy when they detect changes.

Q: Can I scale later?
A: Yes, upgrade from free to paid tiers whenever you need more resources.
